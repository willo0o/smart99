# 智慧工务巡检作业管理系统 · 演示网站

> Smart Works Inspection Management System — Frontend Demo

---

## 一、项目背景

本项目是一套**通用工业工务巡检作业管理系统**的前台演示网站。

需求来源于 `example/` 目录下的原始资料：

- **需求文档**（`example/智慧工务巡检作业管理系统.pdf`）：以某电厂**脱硫脱硝智能控制系统**为具体案例，描述了一套基于深度学习预测模型（BP 神经网络 / LSTM）的工业过程控制平台，涵盖数据采集、特征工程、模型训练与评估、频率自适应调节（含 PID 控制）、模型接口封装、上线部署等完整闭环。
- **界面参考截图**（`example/1.png ~ 6.jpg`）：6 张来自真实系统的运行截图，展示了实时工艺流程图、数据分析、告警管理、控制日志等核心页面的视觉风格与交互模式。

**建设目标**：在不依赖任何后端的前提下，实现一个可在浏览器中**流畅运行、完整演示**的前台系统，视觉与交互对标真实工业 HMI（人机界面）风格。系统在设计上需**通用化**，不与脱硫脱硝强绑定，可适配任意工业巡检场景。

## 三、设计思路

### 3.1 技术选型

| 维度 | 决策 | 原因 |
|------|------|------|
| 运行方式 | 纯静态，直接用浏览器打开 | 无需 Node/Python 服务，零部署门槛 |
| 框架 | 原生 HTML5 + CSS3 + Vanilla JS | 轻量，无构建步骤，便于演示和交付 |
| 图表库 | ECharts 5.x（CDN） | 工业场景成熟度高，暗色主题支持好，折线/柱状/面积图开箱即用 |
| 图标 | Font Awesome 6（CDN） | 覆盖工业图标，中英文无差异 |
| 工艺流程图 | 内联 SVG | 无图片依赖，矢量，可在 CSS/JS 中直接操控动画和数值 |

### 3.2 页面结构设计

采用**两层结构**：

```
index.html （登录页）
    └── dashboard.html （主应用，单页，弹窗模式）
            ├── 顶部 Header：系统标题 + 机组切换 + 时钟
            ├── 次级导航：工艺控制系统 A / B
            ├── 主内容区：SVG 流程图 ＋ 参数面板
            ├── 底部导航：4 个功能按钮
            └── 4 个全屏弹窗：数据分析 / 模型分析 / 告警管理 / 控制日志
```

弹窗模式而非多页跳转，是因为截图中清晰显示：功能页是在主流程图**之上叠加**打开的，主图始终作为背景层存在。

### 3.3 通用化策略

原始需求以脱硫脱硝为例，但系统需通用。采用以下映射：

| 原始术语 | 通用化后 |
|---------|---------|
| 锅炉 1#/2#/3# | 1#机组 / 2#机组 / 3#机组 |
| 脱硫 / 脱硝 | 工艺流程 A / 工艺流程 B |
| SO₂ 出口浓度 | 出口参数1浓度 |
| NOₓ 出口浓度 | 出口参数2浓度 |
| 吸收塔 / 氨水储罐 | 反应塔 / 储液罐 A、B |
| 风机频率 | 设备频率（Hz）|

这样既保留真实系统的参数结构，又不局限于特定行业。

### 3.4 SVG 工艺流程图设计

这是视觉核心，也是最复杂的部分：

- **设备组件**：储罐（圆柱体用 `rect` + `ellipse` 模拟）、泵（圆形 + 旋转叶片）、塔体（矩形 + 穹顶）、烟囱（梯形 `path`）、阀门（蝶形符号）
- **管道动画**：用 SVG `stroke-dasharray` + CSS `@keyframes` 驱动 `stroke-dashoffset`，形成液体流动视觉效果；不同管道使用正向/反向动画区分流向
- **测量点**：彩色脉冲圆（绿色=正常，红色=告警，黄色=预警），使用 `@keyframes` 透明度循环
- **泵旋转**：SVG `<g>` 元素上 CSS `animation: pump-spin` + `transform-origin` 实现叶片旋转
- **实时数值**：每 2.5 秒 `setInterval` 刷新，在基础值上施加 ±5% 随机扰动，模拟真实采集抖动

工艺 A（原料→反应→排放）和工艺 B（稀释→储液→处理→排放）分别建模，通过 CSS `opacity` + `pointer-events` 切换，避免 DOM 重建的性能损耗。

## 四、工作过程

### 阶段一：需求探索

1. 读取 `example/` 下的 PDF 文档，解析 8 大实施模块（需求确认→数据采集→模型设计→评估→调优→频率控制→接口封装→部署）
2. 逐张读取参考截图，提炼视觉风格、页面布局、组件细节、数据字段

### 阶段二：规划设计

制定实现方案，明确：
- 文件结构（4+1 个主文件）
- 设计系统（CSS 变量体系）
- 数据仿真策略
- SVG 组件的绘制方式
- 弹窗交互模式

### 阶段三：逐文件实现

按依赖顺序实现：

```
css/style.css  →  js/data.js  →  js/charts.js  →  js/app.js
      ↓                                                   ↓
  index.html                                      dashboard.html
```

每个文件写完即验证语法（`node --check`），确保无低级错误。

### 阶段四：集成验证

在浏览器中打开 `index.html`，逐项检验：
- 登录跳转
- 实时动画（流程图数值刷新、管道流动、泵旋转）
- 机组切换（参数面板更新）
- 工艺切换（SVG 视图切换）
- 4 个弹窗（打开/关闭/Esc键关闭）
- 图表渲染（查询 → 加载动画 → 折线图显示）
- 分页功能（告警/日志翻页）

## 五、功能模块说明

### 5.1 登录页（index.html）

- Canvas 粒子网络动画（120个粒子，连线阈值110px）
- 任意账号密码均可登录（演示模式）
- 扫描线动画叠加在登录卡片上

### 5.2 主仪表板（dashboard.html）

| 区域 | 内容 |
|------|------|
| 顶部 Header | 系统名称、机组切换（1#/2#/3#）、实时时钟（1s刷新） |
| 次级导航 | 工艺A/B切换、系统状态指示灯 |
| 主内容区 | SVG 工艺流程图（实时动画）+ 右侧参数面板 |
| 参数面板 | 出口参数设定/目标/瞬时/预测值、预警上限、总量、可输入设定值 |
| 底部导航 | 4个功能按钮，告警管理按钮有红点提示未处理告警 |

### 5.3 数据分析弹窗

**模型分析标签页：**
- 筛选条件：机组 + 日期
- 双图联动：出口参数1/2浓度折线图（实时值 vs 预测值）
- ECharts 峰值区间高亮（模拟异常时段）
- 右侧 RMSE/MAE 指标展示

**对标阶分析标签页：**
- 8个参数可选（下拉切换立即重绘）
- Y轴归一化（0-1），模拟合规性对标
- 悬停显示时间戳+数值信息框

### 5.4 模型分析弹窗

- 4个统计卡片（版本/精度/响应时间/数据量）
- 11个历史版本表格（V1～V11，含训练时间、RMSE、MAE、状态）
- 版本误差对比柱状图（双系列：RMSE + MAE）

### 5.5 告警管理弹窗

- 5维筛选（车间/工序/告警点位/告警类别/日期）
- 15条仿真告警记录（含离线/超限/数据异常/通信故障）
- 未恢复告警用红色标注
- 分页（每页8条，共2页）

### 5.6 控制日志弹窗

- 4维筛选（车间/工序/告警点位/日期）
- 27条仿真操作记录（含设备编码和工艺名称）
- 显示开始时间、结束时间、总耗时
- 分页（每页14条，共2页）
